# MedAssist AI — Demo Script (CS 595 Final Presentation)

**Duration:** 6–8 minutes  
**Demo accounts:** See "Setup" below  
**URL:** http://localhost:5173

---

## Pre-Demo Setup (5 min before)

1. Start the server: `cd medassist/server && npm run dev`
2. Start the client: `cd medassist/client && npm run dev`
3. Open [http://localhost:5173](http://localhost:5173) in Chrome (full screen)
4. Keep a sample blood report PDF ready: `pdf_report/iron_deficiency_anemia.pdf`
5. Open a second tab for the Doctor flow

**Demo credentials:**
| Role | Email | Password |
|------|-------|----------|
| Patient | patient1@demo.com | password123 |
| Doctor | doctor1@demo.com | password123 |

## Scene 1 — Landing & Registration (45 sec)

> "MedAssist AI is a full-stack medical informatics platform built for CS 595. It demonstrates three autonomous AI agents that call real medical APIs — FDA, NIH RxNorm, and ICD-10 lookup — to assist both patients and doctors."

- Show the landing / login page
- Click **Register** → fill out a new patient account (or use demo credentials)
- After login, you land on the **Patient Dashboard**

> "The dashboard shows uploaded reports, health scores, and daily wellness tips."

---

## Scene 2 — Symptom Intake Wizard (60 sec)

> "The first thing a new patient does is describe their symptoms. This feeds our Diagnostic Agent."

- Click **Upload Report** in the navbar → Or navigate to **/patient/intake**
- **Step 1:** Age 35, Male, 82 kg, 178 cm, Blood Group O+
- **Step 2:** Existing conditions: `Diabetes`, Allergies: `Penicillin`, Current meds: `Metformin`
- **Step 3:** Select symptoms: **Fatigue** (severity 8, 3 weeks), **Increased Thirst** (severity 7, 3 weeks), **Frequent Urination** (severity 7, 3 weeks), **Blurred Vision** (severity 5, 1 week), **Weight Loss** (severity 4, 1 month)
- Click **Get AI Diagnosis**

> "Watch the live agent steps — it's actually calling ICD-10 lookup APIs at the NIH in real time."

---

## Scene 3 — Disease Results + Blood Test Recommendations (60 sec)

> "The Diagnostic Agent returned the top 5 differential diagnoses with ICD-10 codes and probability scores."

- Show the **Results page**: disease cards with ICD codes (E11, E14, etc.), probability bars, matched symptoms
- Click the top disease (Type 2 Diabetes Mellitus, ~E11)
- Click **Get Recommended Blood Tests**

> "It now calls our Blood Test Recommendation Agent — here are the specific tests ordered with urgency levels."

- Show **Tests page**: HbA1c (essential), Fasting Glucose (essential), CBC, Lipid Panel, Kidney Function
- Click **Upload Blood Report →**

---

## Scene 4 — Blood Report Upload + OCR (45 sec)

> "The patient uploads their lab report — PDF or image. We use Gemini Vision for OCR extraction."

- Drag-and-drop `iron_deficiency_anemia.pdf` onto the upload zone
- Click **Analyze Report**
- Watch the extracted values table appear: Hemoglobin, HCT, MCV, MCH, etc. with status badges

> "Every parameter is extracted, compared against reference ranges, and flagged as normal, high, low, or critical."

- Click **Get AI Analysis →**

---

## Scene 5 — Analysis Dashboard (90 sec)

> "This is the core of the system — the Blood Report Agent runs a full agentic loop calling the FDA drug database and RxNorm for drug interactions."

- Show the **Agent Status Panel** running live (tool call steps streaming in via SSE)
- Once complete, walk through each section:

**Section 1 — Summary:**
> "Overall assessment, root cause, and complexity rating."

**Section 2 — Abnormal Parameter Report (ParameterProgress):**
> "Each abnormal parameter gets a clinical card with a gauge showing where the value falls relative to the normal range — the dot is the patient's value, the green zone is normal."

**Section 3 — Risk Scores:**
> "Composite clinical risk scores for kidney, liver, and cardiovascular health — generated by our Risk Scoring Agent using established clinical models."

**Section 4 — Treatment & Medications:**
> "FDA-verified medication suggestions with dosage, frequency, contraindications — never a prescription, always educational."

**Section 5 — Follow-Up Schedule:**
> "Personalized recheck schedule based on which values were abnormal."

**Section 6 — Doctor Referral:**
> "If 3+ critical values or a drug interaction is found, the system flags for immediate physician consultation."

- Click **Find a Doctor →** (or show the map tab)

---

## Scene 6 — Doctor Finder Map (30 sec)

> "The map auto-detects location and pulls real-time clinic/hospital data from OpenStreetMap."

- Show the Leaflet map with doctor markers
- Click a marker → popup with name, specialty, distance, phone, directions link
- Show the specialty filter and radius selector

---

## Scene 7 — Doctor Dashboard + Assist Agent (75 sec)

> "Now switching to the Doctor role. Doctors use MedAssist to check if they've missed any essential diagnostic tests."

- Switch to the second browser tab, login as `doctor1@demo.com`
- Show the **Doctor Dashboard**: session history, stats row (total sessions, urgent count)
- Click **New Patient Assist**

**Fill in the patient case:**
- Age: 52, Male, Chief complaint: "Chest pain and shortness of breath for 3 days"
- Symptoms: "Chest pain, dyspnea, fatigue, bilateral leg swelling"
- Known conditions: "Hypertension on lisinopril, 20-pack-year smoking history"
- Existing tests checked: CBC, BMP

- Click **Get AI Suggestions**

> "Watch the Doctor Assist Agent run — it calls ICD-10 lookup to confirm the clinical context, then compares standard cardiac workup tests against what's already ordered."

- Show the **results table**: Troponin I (critical), BNP (urgent), Chest X-Ray (urgent), ECG (essential)

- Click **🔍 View Agent Log** on a past session
> "Every tool call is logged — this is the audit trail your professor can inspect to verify the AI is actually calling real external APIs, not just generating text."

---

## Scene 8 — Wrap-Up (30 sec)

> "To summarize: MedAssist AI demonstrates a full medical informatics pipeline — symptom intake, AI diagnosis with ICD-10 verification, blood report OCR, FDA-verified medication planning, risk scoring, and doctor-side diagnostic gap analysis. All using free public APIs and an ensemble of AI providers for higher accuracy."

**Key technical highlights to mention:**
- Multi-provider ensemble: 5 AI providers run in parallel → consensus judge merges results
- Real tool calls: NIH ClinicalTables (ICD-10), OpenFDA, RxNorm
- SSE streaming: live agent step updates via Server-Sent Events
- Full audit trail: every agent run logged to `agent_logs` table

## Backup Plan

If the AI API is rate-limited during the demo:
1. Show a previously analyzed report from the dashboard (click **View Analysis**)
2. The analysis is cached — no re-run needed
3. The integration test output (`node tests/integration.js`) shows all 25 tests passing — use that as proof of end-to-end correctness
